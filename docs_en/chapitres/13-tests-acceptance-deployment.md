# Tests / Acceptance / Deployment

Use the 4D project mode to save the source code of your application as a
text file, allowing the use of a version control system.

You can then use a revision control system (like Git) but also a hosting
service for the source code of your 4D application (like GitHub).

Do not deploy your database without first testing it in its future
environment or in a similar environment (hardware, identical version of
4D, etc.) and especially in its future conditions of use (with the same
number of simultaneous users using user test scenarios).

Stressing your database to find out its limits, undetectable by the
development team, will save you a lot of trouble in Production and will
allow you to optimize the most used features.

Testing and acceptance are the keys to a successful deployment!
