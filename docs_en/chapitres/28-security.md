# Security

Security is an important and fundamental topic for a database or
enterprise solution system. This chapter provides an overview of how 4D
protects your data. In fact, security is data protection. And data
protection is a broad area. Data must be protected from unwanted access,
but also from loss. This is an important fact, because most users only
think about protection from unauthorized users, not protection for
events such as power failure, damaged hard disk, accidental data
modification and so on.

Security and data protection is a very broad area: it starts with user
authentication, goes through external access (like web or SQL), unwanted
code execution (SQL injection, script inspection attacks), then security
updates, backup and more.
